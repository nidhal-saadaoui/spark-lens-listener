How it works

Three steps, no infrastructure required.

1

Attach the listener

Add spark.extraListeners=...SparkLensListener and spark.sparklens.output=text to your spark-submit command or spark-defaults.conf. Nothing else needed.

Heads-up: spark.extraListeners= replaces any existing listener list. Use a comma to append: OtherListener,SparkLensListener. For batch applications only — Structured Streaming results will be misleading.

2

Run your job normally

spark-lens listens to every task, stage, job, and SQL execution as they complete — zero overhead on the hot path. It only processes data after the application ends.

3

Read the report

At application end, a prioritized list of issues is printed to driver stdout or written to a file (or both). Every issue includes a description, a concrete recommendation, and ready-to-paste config or code fixes.

Report output

Three formats: text for driver stdout, HTML for humans, JSON for CI parsers and dashboards.

HTML report — interactive, collapsible issues with stage/job links

Issues are grouped by type — e.g. Disk Spill in Stage 3 [+4 more stages] — so the report stays readable on large pipelines. Click any issue to collapse it.

What it detects

18 analyzers covering skew, memory, I/O, query planning, configuration, and reliability.

SkewAnalyzerskew
p95/p50 task duration ratio > 3× or top-5% tasks hold > 25% of stage time. Distinguishes shuffle skew from input skew with different recommendations.
SpillAnalyzerspill
Total disk spill > 100 MB (Warning) or > 1 GB (Critical). Disk I/O is 10–100× slower than RAM.
JoinAnalyzerjoin
Broadcast disabled on SortMergeJoin, broadcast threshold ≥ 1 GB (driver OOM risk), or ≥ 4 shuffle exchanges in one query.
GcAnalyzergc
GC time > 10% (Warning) or > 20% (Critical) of executor run time. GC pauses can cause heartbeat timeouts.
CacheAnalyzercache
Same RDD scanned in ≥ 2 jobs without caching — the full upstream lineage is re-executed each time.
PlanAnalyzerplan
CartesianProduct (quadratic), Window without PARTITION BY (single-partition serial), round-robin repartition, missing CBO row count statistics.
ConfigAnalyzerconfig
AQE disabled, Java serializer, default shuffle partitions (200), low executor memory overhead.
SmallFilesAnalyzerio
Input avg < 64 MB/task with majority of tasks reading tiny files — too many map tasks, high scheduling overhead.
OutputSmallFilesAnalyzerio
Output avg < 64 MB/task — each task writes one file, so many tasks mean many small files that slow every downstream job.
ShuffleLocalityAnalyzerio
> 70% of shuffle bytes read remotely — cross-rack or cross-AZ reads saturate shared network bandwidth.
DriverBottleneckAnalyzerio
collect() result > 50 MB back to driver, or CollectLimit / TakeOrderedAndProject in the SQL plan.
CpuEfficiencyAnalyzerio
CPU utilization < 20% of executor run time — most core time is I/O wait, shuffle network, or JVM overhead.
PreemptionAnalyzerpreemption
Executor lost mid-job or task kill rate > 5% per stage — indicates resource contention or container eviction.
SpeculationAnalyzerconfig
Speculative tasks actively firing — treating the symptom (slow tasks) instead of the cause (data skew).
StageFailureAnalyzerreliability
Stage retried (attempt > 0) with failure reason, or task failure rate > 5% in a stage.
MemoryPressureAnalyzerreliability
GC > 10% and disk spill > 100 MB co-occurring in the same stage — the executor heap is genuinely undersized.
StageParallelismAnalyzerio
Stage tasks < 50% of available executor cores on a stage > 10s — most cores idle, wall time longer than necessary.
LongStageAnalyzerreliability
Stage duration > 5× the median stage duration in its job — one stage serializes all downstream stages.

Configuration

All settings are optional. No required configuration.

PropertyDefaultValuesDescription
spark.sparklens.outputoffoff · text · json · htmlOutput format. off is silent unless fail.on is set.
spark.sparklens.report.path(stdout)local path, hdfs://, s3://…Write report to a file instead of stdout. Supports any Hadoop-compatible filesystem.
spark.sparklens.fail.on(none)critical · warning · infoThrow RuntimeException at app end if issues at this severity or above are found.

Permanent cluster configuration

Add to spark-defaults.conf on every node — every job gets analyzed automatically, no per-job flags needed.

spark.extraListeners=com.github.saadaouini.sparklens.SparkLensListener spark.sparklens.output=text spark.sparklens.fail.on=critical

CI quality gate

Fail the Spark job itself when critical issues are detected — no external tooling required.

Fail on critical

# CI fails automatically — exit code non-zero spark-submit \ --packages com.github.saadaouini:spark-lens_2.12:0.2.0 \ --conf spark.extraListeners=...SparkLensListener \ --conf spark.sparklens.fail.on=critical \ myJob.jar

Save JSON report as CI artifact

# Parse the JSON in a subsequent step spark-submit \ --conf spark.sparklens.output=json \ --conf spark.sparklens.report.path=/tmp/report.json \ myJob.jar # health_score, issue_count, issues[].severity…
